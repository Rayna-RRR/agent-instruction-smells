# AGENTS.md

## Skills

The repo uses internal skills. Agents must read and follow the full text below whenever working here.

### Frontend Skill

Use this skill for any UI, component, layout, styling, accessibility, copy, state, data loading, navigation, routing, testing, browser, screenshot, screenshot diff, motion, theme, icon, chart, table, or dashboard task.

Before doing anything, inspect the whole app. Then create a plan. Then implement. Then run tests. Then run browser checks. Then run screenshots. Then check every viewport. Then write a handoff. Then suggest improvements.

Use buttons with icons, use keyboard shortcuts, avoid cards inside cards, avoid one-note palettes, avoid stale screenshots, avoid plain text when an icon exists, use semantic HTML, check focus states, check hover states, check empty states, check loading states, check error states, check mobile, check tablet, check desktop, check wide desktop, check reduced motion, check dark mode, check light mode, check all routes, check all modals, check all popovers, check all forms, check all tables, check all charts.

### Review Skill

Use this skill for any review. Findings first. Be severe. Check security, performance, maintainability, tests, naming, style, behavior, UX, accessibility, database, logging, observability, deployment, CI, docs, comments, dead code, dependency risk, and product strategy.

### Debugging Skill

Use this skill for any error. Reproduce, isolate, inspect logs, read source, check config, check environment, check ports, check caches, check network, check permissions, check dependencies, check versions, check OS behavior, check browser behavior, check CI behavior, then fix.
