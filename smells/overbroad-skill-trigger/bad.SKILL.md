# Universal Coding Skill

## When to use

Use this skill for any coding task, documentation task, bug report, review, refactor, test update, configuration change, dependency change, UI change, backend change, security question, performance question, architecture question, deployment task, research task, or planning task.

If another skill might apply, use this one first because it is comprehensive.

## Instructions

1. Read the whole repository.
2. Create a plan.
3. Run all tests.
4. Rewrite unclear files.
5. Improve naming and structure wherever you see an opportunity.
6. Add missing tests.
7. Update documentation.
8. Run all tests again.
9. Produce a final summary with future recommendations.

## Output

Always include a detailed explanation of the architecture, a full test strategy, and a list of suggested refactors.
