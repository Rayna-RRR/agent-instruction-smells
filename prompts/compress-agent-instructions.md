# Prompt: Compress Agent Instructions

Compress the repository's agent instruction file into practical guidance for coding agents.

Do not remove useful project-specific rules. Do not change product behavior. Do not add a new config system, generator, dependency, or CLI.

Please:

1. Keep current commands, important paths, local conventions, and high-risk areas.
2. Remove stale history, copied docs, broad philosophy, old commands, duplicated rules, and generic advice.
3. Replace long background sections with short links when the linked docs exist.
4. Resolve contradictions with explicit precedence.
5. Keep verification guidance risk-based instead of requiring every check for every task.
6. Keep the final file short enough to scan before coding.

After editing, provide:

- A short summary of what changed.
- A list of commands that were verified against project files.
- Any commands or paths that still need human confirmation.
- What was intentionally left out and why.
